# UI Polish Pass（第十一轮：Cat Draft）

## 1) 审计结论：当前最影响猫猫主题一致性的项

1. 历史文案中仍残留 Orchard/Harvest 表达。
2. 主题配置仍需要统一到 cat-draft 语义，供开发/策划/美术协同。
3. 占位视觉仍非正式猫猫素材，必须明确“草案态”而非完成态。

## 2) 本轮低风险可清理项（已做）

- 统一产品表达为 `Lucky Pop · Meow` 主标题体系。
- Login/Level/HUD/Pause/Result 的核心文案切换为猫猫主题语义。
- 主题配置明确 A~F/BIRD 对应的猫猫角色命名。
- 保留占位属性说明，避免误判为已完成正式换皮。

## 3) 仍保留的问题（诚实边界）

- 棋子造型与障碍造型仍是占位资源。
- 特效与动画资产未进行正式猫猫化。
- UI 背景与按钮皮肤尚未完成美术统一。

## 4) 需要正式素材或 Editor 手工处理的事项

1. 7 套猫猫棋子素材与动画帧导入。
2. 障碍（ice/crate/chain）猫猫主题视觉替换。
3. Login/Level/Result 背景、按钮、图标统一。
4. 在 Cocos Editor 中替换资源引用并回归验证（保持 `aniPre` 顺序不变）。

## 5) 本轮定位

本轮是“猫猫主题迁移准备版（Cat Draft）”，用于统一表达、命名与配置，
不是正式猫猫美术完成版。
