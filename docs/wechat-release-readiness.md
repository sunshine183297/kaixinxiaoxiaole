# WeChat Mini Game Release Readiness（提测准备审计）

> 范围：当前仓库的“可提测版本”准备度，不等于“正式上线版本”完成度。

## 1) 当前已具备（对提测有帮助）

- 基础流程完整：`Login -> Level Select -> Game -> Result`。  
- 有本地进度存档能力（星级）与基础设置存档能力（音量）。  
- 暂停 / 继续 / 返回关卡 / 下一关等核心交互已具备。  
- 前 1~10 关已有连续曲线（含解锁门槛和星级公式）。  
- UI 已有产品化语气与占位式果园主题表达（非正式美术）。

## 2) 本轮低风险整理（已做）

- 集中本地存档 key 到 `assets/Script/Utils/StorageKeyConst.ts`。  
  - `selectedLevelId`
  - `levelProgress`
  - `volume`
- 将分散的本地 key 引用统一到常量：  
  - `LevelSession.ts`（选关）  
  - `LevelProgress.ts`（星级进度）  
  - `GameController.ts`（音量设置）
- 清理一处明显开发残留：`LoginController.ts` 未使用的 `assetManager` 导入。

## 3) 当前距离“可提测版本”还缺什么

### A. 代码层可继续补（低风险）
1. 增加“清档入口”（仅测试包可见）便于 QA 回归。  
2. 增加统一版本号/构建号常量，写入启动日志或设置页。  
3. 增加关键流程埋点占位（开始关卡/胜负/退出）接口层（不接真实后端也可先留结构）。

### B. 必须在 Cocos Editor 手工处理
1. 构建配置核对（微信小游戏平台参数、分包策略、压缩策略）。  
2. 各场景在不同分辨率和安全区下的 UI 对齐复查。  
3. 资源引用检查（避免无用资源打包、确保首屏必需资源可快速加载）。

### C. 必须在微信开发者工具 / 发布流程处理
1. 真机性能与帧率检查（中低端机）。  
2. 包体体积、首次加载时长、内存峰值检查。  
3. 提审所需信息（类目、隐私合规、测试账号与说明）。

### D. 必须等待正式美术素材
1. 棋子正式水果素材与动画。  
2. 障碍/特效正式素材。  
3. Login/Level/Result 背景与按钮皮肤统一升级。

## 4) 诚实边界

- 当前适合定义为：**可提测原型版（gameplay + flow 基本可测）**。  
- 仍不应定义为：**正式上线版**（缺正式美术、缺真机发布链路验证、缺平台流程完备性）。
