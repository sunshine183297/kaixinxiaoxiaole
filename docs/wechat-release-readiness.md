# 猫猫消消乐 - 微信小游戏提测准备

> 范围：当前仓库的"可提测版本"准备度，不等于"正式上线版本"完成度。

## 1) 当前已具备

- 完整游戏流程：`Login → Level Select → Game → Result`
- 猫咪主题全套中文 UI 文案
- 6 种猫咪品种棋子 + 1 个金色喵星人特殊棋子
- 20 个关卡（猫咪主题命名），包含步数模式和时间模式
- 本地进度存档（星级）与设置存档（音量）
- 暂停 / 继续 / 返回关卡 / 下一关等核心交互
- 微信小游戏平台兼容层（`WechatCompat` 工具类）
- 微信分享、震动反馈、广告位预留接口
- 微信小游戏构建模板（`build-templates/wechatgame/`）
- 集中的本地存档 key 管理（`StorageKeyConst.ts`）

## 2) 微信小游戏构建步骤

1. 在 Cocos Creator 3.8.6 中打开项目
2. 菜单：`项目 → 构建发布`
3. 选择平台：**微信小游戏**
4. 填写 AppID（需在微信公众平台注册小游戏账号获取）
5. 更新 `build-templates/wechatgame/project.config.json` 中的 `appid` 字段
6. 点击构建
7. 使用微信开发者工具打开构建产出目录
8. 真机预览调试 → 上传 → 提交审核

## 3) 距离"可提测版本"还需完成

### A. 代码层（低风险）
1. 增加"清档入口"（仅测试包可见）便于 QA 回归
2. 增加统一版本号/构建号常量
3. 增加关键流程埋点占位（开始关卡/胜负/退出）

### B. 必须在 Cocos Editor 手工处理
1. 替换猫咪正式美术资源到预制体
2. 构建配置核对（分包策略、压缩策略）
3. 各场景在不同分辨率和安全区下的 UI 对齐复查
4. 资源引用检查（避免无用资源打包）

### C. 必须在微信开发者工具处理
1. 真机性能与帧率检查（中低端机）
2. 包体体积、首次加载时长、内存峰值检查
3. 提审所需信息（类目、隐私合规、测试账号与说明）

### D. 美术素材
1. 6 种猫咪品种的正式棋子美术与动画
2. 金色喵星人特殊棋子素材
3. 障碍物素材（冰冻鱼骨、纸箱子、毛线球）
4. Login/Level/Result 背景与 UI 皮肤统一升级

## 4) 当前状态评估

- **当前定位**：可提测原型版（gameplay + flow 基本可测，猫咪主题文案完备）
- **距离正式上线**：需正式猫咪美术资源、真机发布链路验证、微信平台合规审查
